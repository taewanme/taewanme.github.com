OCI 관리 모바일 앱(Mobile App)

November 12, 2020 Cloud

OCI(Oracle Cloud Infrastructure) 공식 모바일 애플리케이션과 안드로이드 버전과 IOS 버전으로 공개되었습니다.(2020/11/03) OCI 모바일 앱을 이용하여 OCI 리소스 탐색, 리소스 사용량 확인 및 상태 모니터링을 할 수 있습니다. 또한 알람(Alarm)을 설정하여 OCI 자원 상태를 통지받을 수 있습니다. 추가로 이 앱을 이용하여 사용량 청구 및 리소스 제한 정보를 확인할 수 있습니다.

공식 과련 정보는 다음 링크로 확인할 수 있습니다.

  • OCI 공식 메뉴얼(OCI Mobile)
    • https://docs.cloud.oracle.com/en-us/iaas/Content/GSG/Concepts/mobile.htm
  • OCI 공식 블로그
    • https://blogs.oracle.com/cloud-infrastructure/oracle-cloud-infrastructure-mobile-app-anytime-anywhere
  • Google Play - OCI Mobile App
    • https://play.google.com/store/apps/details?id=com.oracle.oci.console.ocimobile&hl=en_US&gl=US
  • Apple Store - OCI Mobile App
    • https://apps.apple.com/us/app/oracle-cloud-infrastructure/id1534250806

OCI 모바일 앱은 다음과 같은 기능을 제공합니다

  • 리전의 Compartment 별로 배포 자원(리소스) 탐색
  • 특정 리소스 검색
    • Compute VM, ATP, ADW, Load Balancer, Block Volume, Object Storage
  • 리소스 상세 속성 확인
  • 리소스 상태 모니터링 (CPU 사용량, 메모리 및 디스크 I/O 메트릭 )
  • OCI 알람
  • 자원 제한 설정 확인

아이폰용 모바일 앱

아이폰용 OCI 모바일 앱을 설치해 보고 살펴보겠습니다. 애플 앱스토어에서 앱을 설치하고 OCI Trial 계정에 로그인하여 앱 구성을 살펴보겠습니다. 우선 애플 앱 스토어에서 “OCI” 혹은 “Oracle Cloud Infrastructure“를 키워드로 검색한 앱을 설치합니다.

설치를 마치고 앱을 실행하면 다음과 같은 앱 화면이 출력됩니다. 앱 화면에 아래 Sign in 버튼을 누르면 다음 페이지로 이동합니다.

앱을 처음 시작할 경우, 다음과 같은 라이센스 동의 화면이 출력됩니다. 화면 하단의 “I Agree”를 누릅니다

로그인할 OCI 계정 명(테넌트 이름)을 입력하고 Contitnue 버튼을 누릅니다.

IDCS 사용자 ID와 OCI IAM 사용자 ID 중에 어떤 ID로 로그인할 것인지를 선택합니다. IDCS 사용자 ID를 사용할 경우, Identity Provider를 선택하고 Continue 버튼을 누릅니다. Identity Provider는 일반적으로 기본 값을 사용합니다.

OCI IAM 사용자 ID를 사용하는 경우라면 아래 “Oracle Cloud Infrastructure Direct Sign-in” 탭을 클릭하고 로그인 합니다.

현재는 IDCS 사용자 ID를 사용하여 진행합니다.

IDCS 사용자 ID와 패스워드를 입력하고 로그인합니다.

로그인하면 앱의 홈 화면에서 지난 24시간의 알람과 결제 정보 및 검색 가능한 리소스 유형을 확인할 수 있습니다.

알람 탭을 누르면 알람 목록을 조회하고 알람 상세 정보를 확인 할 수 있습니다.

리소스 탭을 누르면 Compartment 별로 프로비저닝된 자원을 확인하고, 자원 별로 상세 정보 및 모니터링 상태를 확인할 수 있습니다.

Limit 텝을 누르면 현재 계정에 대한 리전별 자원 유형별 자원 허용 설정을 확인할 수 있습니다.

OCI 모바일 앱 제한 사항

이번에 공개된 모바일 앱은 자원 배포, 시작, 정지, 제거 등 클라우드 자원을 제어하는 기능은 빠져 있습니다. 클라우드 자원 모니터링과 관리 용도에 중심을 두고 있습니다. 자원을 직접 제어하는 부분은 보안 이슈와 관련되기에 신중을 기하고 있으며 향후 추가 기능으로 확장할 계획이라고 합니다.
