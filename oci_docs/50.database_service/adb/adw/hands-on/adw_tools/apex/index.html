    APEX 서비스

  • 문서 생성일: 2020-03-20 08:54:32 +0900 KST

    • Oracle Application Express(APEX)는 어디든 배치할 수 있는 안전하고 확장성을 갖춘 엔터프라이즈 앱을 구축 로우코드 개발 플랫폼입니다. ADW는 자체 서비스로 APEX를 제공합니다. 이 문서에서는 APEX 개발 환경 접속 방법을 정리합니다.

    이번 문서는 오라클 클라우드 Free Tier 계정에 ADW 인스턴스를 만들었다는 전제로 진행합니다. 현재 ADW 인스턴스가 없다면, 다음 문서를 참조하여 ADW 인스턴스(DemoADW)를 만드시기 바랍니다.

    • ADW 인스턴스 생성

    이번 문서에서는 ADW에서 제공하는 APEX 서비스에 대하여 접속 방법과 간단한 예제 애플리케이션 개발 절차를 소개합니다. 다음과 같은 목차로 진행 됩니다.

    ADW 제공 서비스: APEX

    ADW 인스턴스 상세 페이지에서 Tools 텝을 통해서 Oracle Application Express를 접근할 수 있습니다.

    APEX 서비스 접근

    ADW 인스턴스 상세 페이지에서 Tools 텝을 통해서 Oracle Application Express를 접근할 수 있습니다. Oracle APEX 버튼을 클릭하면 Oracle APEX 서비스 페이지가 오픈됩니다.

    처음에 Oracle APEX 버튼을 클릭하면 ADW 인스턴스의 admin 계정 비밀번호 입력화면이 출력됩니다. 비밀 번호를 입력하고 “Sign in to Administration” 버튼을 클릭하면 Workdspace 생성 페이지로 이동합니다.

    Workspace 생성

    Create Workspace 버튼을 클릭하고 다음 페이지로 이동합니다.

    Workspace를 생성하기 위해서 다음 3개자 항목의 설정 값을 입력합니다.

    설정항목 설정값 설명
    Database User APEXUSER 새로운 사용자 등록시 Database User가 생성됨
    Password Welcome123456 비밀번호 설정
    Workspace Name APEXUSER Database User와 같은 값 설정

    설정값을 입력하고 Create Workspace 버튼을 클릭합니다.

    오른쪽 위의 사용자 프로파일을 클릭하고 로그아웃합니다.

    로그라웃 확인 페이지가 출력됩니다. Return to Sign in Page 버튼을 클릭합니다.

    Workspace 로그인

    Workspace을 생성할 때 설정했던 값을 입력하고 로그인합니다.

    샘플 데이터 & 애플리케이션 생성

    Workspace 메인 페이지에서 SQL Workspace를 클릭합니다.

    SQL Workspace에서 Data Workshop을 클릭합니다.

    Load Data 창에서 Copy and Paste를 클릭하고 Classic Departments and Employees를 선택하고 Next 버튼을 클릭합니다.

    테이블이 저장될 Database User와 테이블 명을 입력하고 Load Data 버튼을 클릭합니다.

    Load Data 버튼을 클릭합니다.

    샘플 데이터 로딩 마지막 단계로 애플리케이션 생성을 요청합니다.

    예제 애플리케이션 이름을 Employee로 설정하고 Create Application 버튼을 클릭합니다.

    샘플 애플리케이션 실행

    이제 Employee 테이블과 Dept 테이블로 예제 애플리케이션이 만들어졌습니다. Run Application을 클릭합니다.

    Run Application을 클릭하면 Employee 애플리케이션의 로그인 창이 열립니다. 기본 로그인 인증을 위해서 Workspace를 만들때 등록한 Database User 정보를 입력합니다.

    Employee 애플리케이션은 Employee 테이블 기본 검색과 리포트 페이비가 만들어져 있습니다.
