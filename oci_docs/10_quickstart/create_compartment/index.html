    컴파트먼트(Compartment) 추가

    이번 문서는 오라클 클라우드 Free Tier 계정이 생성된 상황을 전제로 합니다. 만약에 현재 오라클 클라우드 계정이 없는 상태라면, 다음 문서를 참조하여 오라클 클라우드 계정을 만들고 진행하시기 바랍니다.

    • OCI FREE TIER 계정 등록

    OCI 사용자가 OCI에서 만드는 대부분의 자원을 생성할 때, 자원을 논리적으로 묶는 컴파트먼트를 지정해야 합니다. Compartment를 통해서 논리적으로 묶인 자원은 모니터링하고 과금하고 청구하고 접근 권한을 설정할 수 있습니다.

    이번 문서에서는 OCI 콘솔에서 컴파트먼트를 등록하는 과정을 소개하며 다음과 같은 목차로 구성됩니다.

    • Compartment 추가
    • 참고문서

    • Compartment 한글화
      • 컴파트먼트를 OCI 콘솔에서는 “구획“으로 번역합니다.

    Compartment란?

    컴파트먼트는 OCI 보안과 접근 제어에 기반이 됩니다. 컴파트먼트는 논리적으로 자원을 묶는 OCI IAM(Identity and Access Management)의 핵심 컴포넌트 입니다. 컴파트먼트는 중첩 디렉터리 구조를 갖습니다. Oracle Cloud 계정인 테넌시(Tenancy)가 만들어지면, 관리자 컴파트먼트인 Root Compartment가 만들어 집니다. Administrator 사용자 그룹에 포함된 OCI User는 Root 컴파트먼트 아래에 컴포넌트를 만들 수 있습니다. 컴파트먼트는 파일시스템의 폴더와 완전히 같은 형태를 갖습니다. OCI에서 모든 자원은 특정 컴파트먼트에 소속됩니다. 사용자 그룹에 컴파트먼트 단위로 접근과 권한 정책을 설정합니다.

    컴파트먼트는 OCI 글루벌 범위를 갖습니다. 컴파트먼트는 특정 리전에 종속적이지 않습니다. 특정 리전에서 만든 컴파트먼트는 모든 리전에서 사용 가능합니다.

    • 사용자 그룹: HR-Admins
    • Compartment: HR
    • 정책: HR-Admins 사용자 그룹에 포함된 모든 사람에게 HR 컴파트먼트에 포함된 모든 자원의 관리 기능을 부여

    다음과 같은 Policy로 위와 같은 접근제어 권한을 설정할 수 있습니다.

    allow group HR-Admins to manage all-resources in compartment HR

    Compartment 추가

    Root 컴파트먼트 아래에 demo 컴파트먼트를 추가할 것입니다. 예제로 사용할 Oracle Cloud의 계정(테넌시, Tenancy), OCI User 및 리전은 다음과 같습니다.

    • 테넌시: meetup101
    • OCI 사용자 ID: oracle.testa1@gmail.com
    • OCI 사용자 그룹: Administrator
    • 리전: 서울(ap-seoul-1)

    현재 사용하는 OCI 사용자는 Administrator 그룹에 소속된 상태입니다. Administrator 그룹에 소속된 사용자 ID는 관리자권한을 갖고 있습니다. 따라서 meetup101 테넌시의 모든 자원과 컴파트먼트에 대한 접근 및 관리 권한을 갖고 있습니다. 따라서 컴파트먼트를 만든 후 별도의 Policy 설정 작업을 하지 않습니다.

    이것은 처음에 빠른 실습과 절차를 단순하게 만들기 위한 조치입니다. 실제 환경에서는 관리자 사용자 ID를 사용하면 않됩니다. 따라서 실무에서는 컴파트먼트를 만들고, 컴파트먼트를 사용하는 사용자에게 적절한 권한 설정을 먼저 해야 합니다.

    Root 컴파트먼트 아래에 demo 컴파트먼트를 생성합니다. 컴파트먼트 관리 페이지 이동 메뉴는 다음과 같습니다.

    (OCI 콘솔 왼쪽 위 햄버그 버튼) Identity Compartments

    컴파트먼트 관리 페이지에서 생성 버튼을 클릭하여 다음 그림과 같이 root 컴파트먼트 아래에 demo 컴파트먼트를 생성합니다.

    위 그림을 통애서 다음과 같이 root 컴파트먼트 아래에 demo 컴파트먼트가 생성된 것을 확인 할 수 있습니다.
